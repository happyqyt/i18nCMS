正在检查版本相关计算属性与 `currentVersion` 的绑定逻辑。


定位到问题：`activeVersion` 依赖 `this.isDraftVersion` 的链式 computed，且 `currentVersion` 与 `v.id` 可能存在类型不一致；版本切换时若 id 无法匹配会导致判断失效。正在重构版本解析逻辑并加固数据。


Vue 未挂载：`PROJECTS.forEach` 在 `PROJECTS` 定义之前被调用，导致 JS 报错。


问题已定位并修复，主要有两类原因：

### 根因 1：JS 运行时错误（最严重）
上次改动里在 `PROJECTS` 定义**之前**就调用了 `PROJECTS.forEach(...)`，导致 `app.js` 加载失败，Vue 无法挂载，模板里的 `{{ }}` 原样显示，所有 computed（`isDraftVersion` 等）都不会生效。

已把初始化移到 `PROJECTS` 数组定义之后。

### 根因 2：版本 ID 匹配不稳健
- `<select>` 的 value 始终是**字符串**，与 `v.id` 比较时可能因类型不一致导致 `activeVersion` 为 `null`
- `activeVersion` 链式依赖 `this.isDraftVersion`，在部分状态下判断不稳定
- 切换系统时 `currentVersion` 未按系统分别记忆，容易出现无效版本 ID

### 修复内容
1. 统一使用 `DRAFT_VERSION_ID` 常量
2. 新增 `findVersionById()` / `validateCurrentVersion()`，用 `String()` 做 ID 比较
3. 重写 computed，避免链式依赖；`canExport` / `canLockVersion` 要求 `activeVersion` 存在
4. 每个系统用 `_viewVersionId` 记住当前查看的版本
5. `<option :value="String(v.id)">` + `@change="onVersionSelect"` 保证下拉切换可靠
6. 启动时为所有版本补全 `locked: false` 默认值
