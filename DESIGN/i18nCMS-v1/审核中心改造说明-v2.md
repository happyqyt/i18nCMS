# 审核中心改造说明 v2.0

## 改造概述

将原来的"我的操作"和"我的审核"两个独立页面合并为一个统一的审核中心页面，简化导航结构，优化用户体验。

## 主要变更

### 1. **页面合并**
- 原来的两个页面：
  - `review/toReview.html` - 我的操作
  - `review/myReview.html` - 我的审核
- 合并为一个新页面：
  - `review.html` - 审核中心（根目录）

### 2. **导航菜单简化**
- **之前**：审核中心是下拉菜单，包含"我的操作"和"我的审核"两个子菜单
- **现在**：审核中心直接是一个链接，点击直接进入审核中心页面

### 3. **页面功能设计**

#### 3.1 筛选表单（顶部）
位于页面顶部的筛选区域，包含4个筛选条件：

1. **发起人**：文本输入框，支持模糊搜索
2. **审核人**：文本输入框，支持模糊搜索
3. **时间范围**：下拉选择
   - 全部
   - 今天
   - 最近7天
   - 最近30天
4. **审核单状态**：下拉选择
   - 全部
   - 待审核
   - 已通过
   - 已驳回
   - 已撤销

两个操作按钮：
- **重置**：清空所有筛选条件
- **查询**：应用筛选条件

#### 3.2 数据表格
显示审核申请列表，包含以下字段：
- 文案 Key
- 所属目录
- 变更类型（新增/修改/删除）
- 发起人
- 审核人
- 提交时间
- 状态（待审核/已通过/已驳回/已撤销）
- 操作

#### 3.3 智能操作按钮
根据申请单的状态、发起人和审核人，动态显示操作按钮：

**规则1：当前用户是发起人 + 状态是待审核**
- 显示"撤销"按钮
- 显示"查看详情"按钮

**规则2：当前用户是审核人 + 状态是待审核**
- 显示"通过"按钮
- 显示"驳回"按钮
- 显示"查看详情"按钮

**规则3：状态是已通过/已撤销**
- 只显示"查看详情"按钮

**规则4：状态是已驳回**
- 显示"查看详情"按钮
- 显示"查看驳回原因"按钮

#### 3.4 统计信息（表格下方左侧）
实时统计当前筛选结果的数据：
- 共计 X 条数据
- 待审核: X 条（橙色图标）
- 已通过: X 条（绿色图标）
- 已驳回: X 条（红色图标）
- 已撤销: X 条（灰色图标）

#### 3.5 分页控件（表格下方右侧）
简洁的分页导航：
```
上一页 1 2 下一页
```

### 4. **移除的功能**
- ❌ 批量操作按钮（批量通过、批量驳回、批量撤销）
- ❌ Tab切换（待审核/已通过/已驳回等标签页）
- ❌ 顶部统计卡片区域

### 5. **保留的功能**
- ✅ 单个审核操作（通过/驳回/撤销）
- ✅ 驳回原因输入和查看
- ✅ 查看申请详情
- ✅ 实时统计
- ✅ 分页功能

## 技术实现

### 筛选逻辑
使用JavaScript实现前端筛选：
- 根据筛选条件动态显示/隐藏表格行
- 实时更新统计数据
- 支持多条件组合筛选

### 权限控制
基于当前登录用户（`currentUser = 'Elin'`）判断：
- 对比申请的发起人，决定是否显示"撤销"按钮
- 对比申请的审核人，决定是否显示"通过/驳回"按钮

### 数据属性
每个表格行包含以下data属性：
- `data-status`：审核单状态
- `data-submitter`：发起人姓名
- `data-reviewer`：审核人姓名

## 用户体验优化

### 优势
1. **操作更简单**：不需要在多个tab或页面之间切换
2. **视图更统一**：一个页面可以看到所有审核相关的信息
3. **筛选更灵活**：通过筛选条件可以快速定位目标数据
4. **统计更直观**：实时显示筛选结果的统计信息

### 使用场景
1. **作为发起人**：筛选"发起人=我"，查看我提交的申请状态
2. **作为审核员**：筛选"审核人=我 + 状态=待审核"，处理待审核的申请
3. **查看历史**：筛选"状态=已通过/已驳回"，查看历史审核记录
4. **综合查询**：组合多个条件，精确定位目标数据
