# user prompt
1.1 项目背景
随着企业全球化业务的拓展，产品、营销内容需覆盖多国多语言市场。传统基于 Excel 或分散文件的模式存在管理混乱、协作低效、发布周期长等痛点。本系统旨在搭建一套集中化、流程化、智能化的多语言文案管理平台，实现文案全生命周期管理，提升从编辑到发布的整体效率。
1.2 建设目标
集中管理：统一管理所有项目文案。
快速交付：一键导出多格式文件，支持一键发布到S3。
资产复用：建立公共文案库，提升高频文案复用率，降低翻译成本。
1.3 适用范围
前端开发所涉及的面对C端的H5页面

### 1. 典型竞品概览

你可以将现有市场大致分为三类：  
- **纯多语言文案管理平台**（面向开发者/PD）：如云词（YunCi）、Phrase / Crowdin / Lokalise 等。 [yundict](https://yundict.com)
- **“国际化翻译 + 文案管理”一体化平台**：如火山引擎 i18n 翻译平台。 [volcengine](https://www.volcengine.com/product/i18ntranslate)
- **多语言 CMS / 内容平台**（偏内容管理）：如 Contentful、Strapi 等多语言 CMS。 [shellednuts](https://shellednuts.blog/blog/multilingual-cms-selection-integration-1754679980110)

以下仅聚焦与你描述功能最接近的前两类竞品。

***

### 2. 云词（YunCi）  
**定位**：面向 App、Web、H5 等场景的多语言文案平台，主打“集中管理 + 开发友好”。 [sspai](https://sspai.com/post/87411)

**主要能力**  
- 提供控制台集中管理多项目、多语种的文案，支持多语言键值对结构。  
- 支持 10+ 种常见格式导入/导出（如 JSON、.strings 等），适合已有工程集成。  
- 提供在线机器翻译（100+ 语种）和人工翻译路线，支持公共词条/语料复用。  

**与你项目的相似点 / 启示**  
- 与你“集中管理所有项目文案”“一键导出多格式文件”的目标高度一致。  
- 可借鉴其“控制台 + CLI 工具”双模式，既方便运营/产品经理配置，也方便前端集成。  

**差异点（你的机会）**  
- 云词更偏通用多端（iOS/Android/Web），对“H5 面向 C 端 H5 页面”场景的营销流程（如活动页、弹窗、运营位）抽象不足，可针对 H5 运营场景做更精细的“位”和“项目”划分。  

***

### 3. 火山引擎 i18n 翻译平台  
**定位**：平台+服务式国际化解决方案，面向企业级多语言产品发布。 [volcengine](https://www.volcengine.com/product/i18ntranslate)

**主要能力**  
- 文案与代码解耦，通过 SDK / i18n 框架收集多端文案，统一管理多语言内容。  
- 支持多语言文案管理、多格式文档翻译、术语库与语料复用，关注翻译质量与一致性。  
- 支持多 H5 页面、小程序、Web 应用等场景，可对接 SaaS 发布/ CDN 等。  

**与你项目的相似点**  
- 与你“快速交付、一键发布到 S3 / CDN”“资产复用”“降低翻译成本”的目标非常接近。  
- 其“规范本地化流程”“前置翻译”“统一管理多端文案”的思路可直接参考。  

**差异点（你的机会）**  
- 火山引擎更偏“平台+翻译服务”的大厂方案，对中小团队或内部 H5 项目来说可能偏重；  
- 你可以围绕 H5 项目与前端工程（如 Vue + i18n）做更轻量、更贴近中国出海团队的 H5 专用工作流。 [juejin](https://juejin.cn/post/7449202839236476955)

***

### 4. Phrase / Crowdin / Lokalise（国际主流本地化平台）  
**定位**：国际主流的本地化协作平台，服务全球化 SaaS、游戏、App 等。 [github](https://github.com/alibaba/kiwi/issues/137)

**主要能力**  
- 强协作工作流：项目经理、翻译、校对、开发各角色在线协作，支持翻译记忆、术语库、审校流程。  
- 与 Git、CI/CD、多种框架（iOS/Android/Web）集成，支持自动拉取/推送文案。  
- 提供多语言导出能力（JSON、YAML 等），部分支持导出到 CDN 或云存储。  

**与你项目的相似点**  
- 与你“全流程管理从需求→编辑→翻译→发布”的建设目标一致。  
- 可借鉴其“状态管理”（待翻译、校对中、已发布）、版本记录、变更对比等产品设计。  

**差异点（你的机会）**  
- 这些工具更偏向“文本本地化工程”，对“营销性 H5 页面文案”（如 Banner、弹窗、活动页副本）的业务场景支持不足；  
- 你可以围绕“营销 H5 + 运营位”做业务抽象，提供更贴近运营的“项目—页面—位”结构，而非仅“key—value”结构。

### 6. 给你项目的建议方向

1. **场景聚焦**：  
   - 与云词、Phrase 等“通用本地化工具”错位，明确主打“H5 C 端营销页面文案”，并引入“项目—页面—位”（Banner/弹窗/按钮文案）等业务维度。

2. **流程深化**：  
   - 借鉴 Phrase/Crowdin 的审批流程与状态管理，但简化为适合 H5 运营节奏的“待编辑→待翻译→已上线”轻量流程。

3. **技术对接友好度**：  
   - 参考火山引擎 i18n 平台的“SDK/集成”思路，提供 H5 工程（如 Vue + vue‑i18n）的对接方案，支持按 H5 页面/项目分包导出 JSON。 [juejin](https://juejin.cn/post/7449202839236476955)

4. **资产复用和成本控制**：  
   - 明确“公共文案库”设计，对高频文案（如通用按钮、免责说明）做标签化，自动替换，减少重复翻译和 S3 内容冗余。
