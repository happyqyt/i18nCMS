# 审核中心页面重新设计说明

## 改造概述

将原有的单一审核中心页面改造为带有三个二级菜单的结构,实现审核流程的细分管理。

## 菜单结构

点击顶部导航栏的"审核中心"后,会出现下拉菜单,包含以下三个子页面:

### 1. 我的操作 (`review/toReview.html`)
- **功能**: 列举当前用户提交的待审核操作列表
- **主要操作**: 
  - 查看我提交的所有审核申请
  - 支持"撤销"操作(仅限待审核状态)
  - 批量撤销功能
  - 查看申请详情
- **统计信息**:
  - 待审核数量
  - 已通过数量
  - 已驳回数量
  - 已撤销数量

### 2. 我的审核 (`review/myReview.html`)
- **功能**: 列举待当前用户审核的申请列表
- **主要操作**:
  - 通过/驳回单个申请
  - 批量通过功能
  - 驳回时需要填写原因
  - 查看申请详情
- **统计信息**:
  - 待我审核数量
  - 今日已审核数量
  - 本周已通过数量
  - 本周已驳回数量

### 3. 审核日志 (`review/reviewLog.html`)
- **功能**: 列举所有审核记录和历史操作
- **主要操作**:
  - 查看完整的审核历史
  - 查看驳回原因
  - 按时间范围筛选(最近7天/30天/90天/全部)
  - 导出日志功能
- **统计信息**:
  - 总审核次数
  - 通过率
  - 平均审核时长
  - 今日审核数量

## 页面特性

### 导航设计
- 所有主页面(`dashboard.html`, `statistics.html`, `version.html`)的顶部导航都已更新
- 审核中心显示为下拉菜单形式,hover时显示三个子菜单
- 当前页面高亮显示

### 交互功能
- **我的操作页**: 支持单个撤销和批量撤销
- **我的审核页**: 支持通过/驳回操作,驳回需填写原因
- **审核日志页**: 支持查看驳回原因的详细弹窗

### 数据展示
- 每个页面都有统计卡片展示关键指标
- 表格展示详细的审核列表
- 支持分页浏览

## 技术实现

### 样式
- 使用 Tailwind CSS 构建响应式界面
- 下拉菜单使用 CSS `group-hover` 实现
- 过渡动画使用 `transition-all` 类

### 图标
- 使用 Iconify 图标库
- 下拉箭头: `mdi:chevron-down`
- 其他功能图标保持统一风格

### JavaScript功能
- 全选/取消全选复选框
- 单个操作处理(通过/驳回/撤销)
- 批量操作处理
- 弹窗管理(驳回原因输入/查看)
- 表单验证

## 使用说明

1. **提交操作后**: 在"我的操作"页面查看审核状态
2. **收到审核任务**: 在"我的审核"页面处理待审核的申请
3. **查看历史**: 在"审核日志"页面查看所有审核记录
