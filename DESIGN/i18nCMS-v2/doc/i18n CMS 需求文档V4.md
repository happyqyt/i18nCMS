# i18n CMS 需求文档V4

# 1\. 项目概述

## 1\.1 项目背景

随着企业全球化业务拓展，传统基于 Excel 或分散文件的文案管理模式面临管理混乱、协作低效、翻译成本高及发布周期长等痛点。本系统旨在建立一个集中化、流程化、智能化的平台，覆盖文案从创建、翻译、审核到发布的全生命周期。

## 1\.2 建设目标

- **集中管理**：打破数据孤岛，统一项目文案管理。

- **快速交付**：支持多格式导出。

- **资产复用**：通过相同文案的复制功能，提升复用率，降低重复翻译成本。

## 3\.3 文案管理

### 3\.3\.1 查

- 表格交互翻页查看

- 支持切换版本查看当前页面的文案

- 若切换到某一版本后当前页面不存在则toast提示

- 搜索

    - 按Key / 文案内容在项目内模糊查询

    - 搜索范围：当前页面、当前项目、所有项目

    - 支持将搜索出来的其他项目的文案复制到当前项目指定页面下（**文案复用**）

### 3\.3\.2 增 / 改

#### **3\.3\.2\.1 手动录入**

- 参考excel表格编辑交互

- 支持批量粘贴

- 支持变量占位符（如 `{user_name}`）

- Key为空则在英文文案输入完成后自动生成Key，支持手动设置Key

- 富文本支持：支持在**弹窗**中用富文本编辑器编辑文案，支持加粗、颜色、超链接等

#### 3\.3\.2\.2 导入

- 页面级操作

- 支持格式：JSON, Excel, CSV。

- 导入Excel or CSV：Excel和CSV中，根据有无文案Key（第一列）进行新增或覆盖

    - 新增模式：自动生成Key，规则见3\.3\.2\.3

    - 覆盖模式：存在相同Key则覆盖，不存在则新增。若遇到没有Key的文案则认为是新增文案，Key走自动生成规则

- 导入JSON：

    - 新增or覆盖：文案Key已存在则覆盖，不存在则新增（使用JSON中的Key进行文案新增）

- 解析出文案，显示文案列表进行预览

    - 按照页面层级显示

    - 用颜色区分新增和覆盖

    - 可以勾选文案选择导入哪些文案

- 注意：

    - 文案覆盖支持跨页面覆盖

    - 新增的文案都归属在当前页面下

#### 3\.3\.2\.3 导出

- 项目级操作

- 导出Excel（csv不支持带样式的文案）

    - 导出Excel时应附带页面截图（若有上传）

- 导出JSON

    - 支持选择是否按照一个语种一个文件的方式导出

- 导出文案范围

    - 支持选择语种（默认全部）

    - 选择文案（树级勾选方式，默认选中全部，勾选页面则选中该页面下的全部文案）

- 不考虑页面这个层级的key

#### 3\.3\.2\.4 唯一标识（Key）

- 自动生成：截取原文案前16字符，空格转下划线 `_`，保留大小写，过滤掉标点符号、特殊字符、emoji

- 冲突处理：若 Key 重复，自动按照\`key\_$\{随机2位字符\}\`格式进行拼接

- 唯一性范围：同项目下唯一

### **3\.3\.3 删**

- 支持单条或批量删除，硬删除

- 支持按语种删除

### 3\.3\.4 其他操作

- 迁移：批量或单个文案，可将指定文案迁移到指定页面

- 复制：批量或单个文案，可选择复制到当前页或指定页面，Key按冲突规则重新生成

## 3\.4 版本管理

> 「我们只需要做一个版本管理就行了，每次更新的文案，觉得这次 ok 了，就打一个版本，之后编辑未打版本的都认为是草稿。」
> 
> 项目规模较小，涉及人员少，审核流程复杂且不必要，版本管理可有效解决文案更新和使用的问题，避免旧文案受影响。
> 
> 

- **核心目标**：可追溯、可对比、可回滚

- **项目级**：版本属于项目级的信息，打版本、回滚、版本查看都是在项目级别上的操作

- **打版本**

    - 研发在认为文案稳定可用时，手动点击“打版本”。

    - 系统生成一个版本号（如 v1, v2），并将当前项目的完整 JSON 字符串存入数据库作为快照。

    - 包含信息：

        - 项目信息（项目名称、项目描述、语种、文案总数、成员列表）

        - 页面信息（页面id，名称、截图）

        - 文案信息（文案id，各语种内容）

- **版本比对**：支持 Diff 视图，高亮展示两个版本间的文案差异。

- **版本查看**：选择特定版本查看页面和文案，不支持修改。

- **版本导出**：选择特定版本进行导出。

- **回滚**：支持一键回滚至历史指定版本。

    - 对比历史版本和当前，Diff视图预览（颜色区分增删改）

    - id级别对比（页面id、文案id）

    - id相同则直接修改，id不同，则进行删除和新增

    - 删除：当前存在但历史版本里不存在的

    - 新增：当前不存在但历史版本里存在的，会生成新的页面id和新的文案id

# 4\. 系统设置与扩展

## 4\.1 语种管理

- **标准规范**：语言码遵循系统预设 List（参考 VeSync Wiki 规范）。

- **校验机制**：导入时强制校验语言码合法性。

## 4\.2 迭代优化（Roadmap）

- **AI 翻译集成**：接入 Google、DeepL、阿里云等 API，实现一键预翻译。

- **从截图识别文案**：根据上传的截图自动识别文案并进行录入。

- **开发者工具**

    - 提供 CLI 工具或插件，支持开发环境下自动拉取最新文案。

    - 提供标准接口，支持工具或命令行导入导出。比如：

        - `GET /api/project/version/latest`：获取最新正式版本数据。

        - `GET /api/project/version`：获取特定版本数据。

        - `POST /api/import`：支持通过接口上传文件（JSON/Excel）更新文案。

- 操作日志记录

- 发布：一键发布到S3，得到文案资源url，项目直接通过script标签引入文案资源

- 审核机制：投入使用后再看情况是否需要
